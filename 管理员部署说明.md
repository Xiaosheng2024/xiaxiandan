# EHX 下线防错程序 - 管理员部署说明

## 环境要求

- Windows 10/11 64 位。
- Microsoft Excel 已安装并能正常启动。
- 现场打印机驱动已安装，Windows 测试页打印正常。
- 程序目录对当前用户可写。

程序默认使用代码生成的 Code128 PNG，不需要安装条码字体，也不需要安装额外的
PDF 转换或静默打印软件。

## 部署

1. 解压 `EHX下线防错_Windows.zip`。
2. 保持 EXE、`config.json`、物料表和模板位于同一目录。
3. 在 `config.json` 中设置 `printer_name`：
   - 留空：使用 Windows 默认打印机。
   - 非空：必须填写 Windows 打印机的完整名称。
4. 确认 `box_scan_count`、产线名称和工位名称正确。
5. 运行 `EHX下线防错.exe`。

## 物料表与每箱数量

`EHX物料号匹配.xlsx` 必须使用以下列：

| 列 | 字段 |
| --- | --- |
| A | 物料条码前缀 |
| B | 物料名称 |
| C | 客户物料号/SAP物料号 |
| D | 每箱数量 |

D 列为空时使用 `config.json` 中的全局 `box_scan_count`；填写时必须是大于 0 的
整数。第一件扫码后，当前箱按对应物料的 D 列数量进行满箱校验。该数量会写入历史
下线单，不会因以后修改 Excel 而改变。

新箱尚未扫描首件时数量显示为 `0/--`，首件识别物料后才锁定该箱数量。

修改 Excel 后，可在主界面打开“物料查看 / 重新导入”并点击“重新导入物料”，
界面会显示新增、更新、禁用数量；也可以重启程序自动同步。

可在 PowerShell 中查看打印机完整名称：

```powershell
Get-Printer | Select-Object Name
```

正式打印使用 Microsoft Excel COM：

```text
printer_name 为空  -> worksheet.PrintOut()
printer_name 非空  -> worksheet.PrintOut(ActivePrinter=printer_name)
```

如果指定打印机不存在或打印失败，程序不会删除 PDF 或同名 XLSX，数据库订单状态会
记录为 `PRINT_FAILED`，GUI 会显示失败原因。修正打印机配置后，可在历史记录中补打。

## 必需的同级文件

- `EHX下线防错.exe`
- `config.json`
- `EHX物料号匹配.xlsx`
- `报交下线单模板.xlsx`

## 验收

1. 执行正常、重复、混料、未配置和格式错误扫码测试。
2. 扫满一箱，确认生成单页 A5 横向 PDF。
3. 检查纸面物料号、客户料号和下线单号条码可扫描。
4. 模拟打印失败，确认 PDF、XLSX 和数据库记录仍保留。
5. 在历史查询中打开 PDF 并执行补打。

## 数据目录

- `data/`：SQLite 数据库。
- `logs/`：运行与错误日志。
- `output/pdf/`：PDF 和同名 XLSX 留档。
- `output/barcodes/`：程序生成的 Code128 PNG。

程序直接使用 SQLite，不依赖外部数据库管理工具。
